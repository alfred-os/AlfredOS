# Gateway Control Plane — Roadmap for Specs B & C (decisions captured, not yet designed)

- **Date:** 2026-06-13
- **Status:** Roadmap / decision-capture. **Not** a buildable spec — Specs B and C each get their own full `brainstorming → spec → plan` cycle when their turn comes. This doc preserves the architecture decisions and the fleet-review findings so that work doesn't restart from zero.
- **Parent:** [Comms-Resume Gateway (Spec A)](2026-06-13-comms-gateway-resume-design.md) ships first and closes #237. B and C open only after A proves out in prod.
- **Source:** two 8-agent `/review-pr` passes on the (now-split) widened design, 2026-06-13.

## The program

The gateway becomes the **mandatory chokepoint for all external network I/O**, with the **core connectivity-free** (no external sockets except via the gateway). Delivered in three gated epics:

- **Spec A — Comms-resume gateway (G0–G5)** — *ships first.* Resumable dial-in transport for the TUI. No adapter hosting, no egress proxy, no connectivity-free claim.
- **Spec B — Adapter-hosting inversion (G6)** — move comms adapters (Discord → Telegram) from daemon-spawned to **gateway-hosted + sandbox-supervised**, so platform sessions survive a core restart (resume generalizes beyond the TUI). Gated on A.
- **Spec C — Egress control plane + connectivity-free core (G7)** — the gateway becomes the sole egress for all outbound I/O (tools + providers); the core loses its external network. ADR-0034 + the PRD §5 invariant rewrite land here. Gated on B.

## Locked maintainer decisions (apply to B/C)

1. **All external network I/O traverses the gateway; the core is connectivity-free** (the program's reason for existing — strengthens PRD goal #8 / §7.1 egress allowlists from policy-gated to structurally-gated).
2. **Provider/LLM egress = forward-proxy with TLS passthrough.** The core's provider SDK targets the gateway as an egress forward-proxy; TLS terminates at the provider, so the gateway enforces the **destination** allowlist + audit + canary-on-destination but **never sees the decrypted prompt**, and native SDK streaming is preserved. This makes "connectivity-free core" structurally true *without* turning the gateway into a prompt-transcript sink. The quarantined extractor's provider call routes through the same forward-proxy (or the dual-LLM split is defeated). *(Resolves the CRITICAL finding all 8 reviewers raised.)*
3. **Two egress modes** (Spec C must define both): (a) **destination-allowlisted forward-proxy / TLS-passthrough** for providers + any egress where the gateway must not see plaintext; (b) **inspecting egress** (frame-relay with a core-DLP-redacted body) for tool egress (web posts, emails) where canary/destination control on the body matters. The single `egress.request{redacted_body}` schema from the draft is insufficient — model both.
4. **Adapter-hosting inversion** (B): the gateway runs the **host role** (the `CommsPluginRunner` — handshake/lifecycle/crash/breaker terminate gateway-local); only **T3 inbound bodies** + `outbound.message` results cross the payload-blind wire to the core. Reuse the existing launcher scrubbed-env + per-adapter bwrap spawn discipline (`_spawn_comms_adapter`).

## Spec B (adapter inversion) — captured findings

- **Who runs the runner** `[comms F1, HIGH]`: the gateway runs the host-side runner and terminates the full plugin lifecycle locally; name which of {handshake, lifecycle, crash, restart, breaker} are gateway-local vs cross-wire.
- **Identity-resolve + rate-limit are ALREADY core-side** `[comms F2, HIGH]` (not adapter-side): `_PreResolutionLimiter`, `_IdentityResolverLike.resolve`, `_emit_binding_request` live in `inbound.py`. They stay core-side, reached over the wire — but add a **gateway-ingress first-line per-platform-id rate gate** so an unbound-user flood is bounded *before* it traverses gateway buffering + the wire (the core limiter becomes defense-in-depth).
- **Credential concentration** `[security F2/F5, HIGH — principal new threat]`: the gateway hosts all platform creds. Mitigation that must be made non-contradictory: keep the **vault + decryption keys in the connectivity-free core**; the core injects the resolved plaintext secret into the **scrubbed child env at adapter spawn** (reusing the launcher discipline) so the gateway forwards an already-substituted opaque path and **never holds a vault key**; per-adapter sandboxed children with no shared cred memory. Testable invariant: *no process holding external network can decrypt more than one platform credential's plaintext.* Corpus: cross-adapter cred-read refused + audited; gateway kernel inspected for plaintext keys → absent.
- **Gateway privilege / image** `[devops F3, HIGH]`: adapter sandbox-supervision pulls `bubblewrap` + the launcher + `cap_add: SETUID` into the gateway image — it is **stable-code but not low-privilege**; reframe the "dumb/stable" language, restate the gateway uid/privilege model, and reconcile the `devops-010` compose invariant (which today asserts only `alfred-core` has SETUID). Security co-sign required.
- **Adapter crash loudness** `[error F4]`: audited crash + bounded-backoff restart + `gateway_adapter_up`/`restarts_total` metric + per-adapter breaker → loud, never silently dark. Mirror the daemon plugin supervisor; migrate (don't drop) the existing Discord adapter test-suite incl. its capability-grant/refusal tests.
- **Setup flag-day** `[devops F5]`: the inversion deletes the `alfred-discord` compose service + moves its secret bind-mount to the gateway, and breaks `bin/alfred-setup.sh`'s `docker compose run --rm alfred-discord verify` path — enumerate the operator migration.

## Spec C (egress control plane + connectivity-free core) — captured findings

- **Provider egress = forward-proxy (decision 2).** Provider responses are **T2/assistant output, not T3** — the "egress-response is T3" rule must carve out providers, or every model token is mis-tagged `[architect NS-1, security F1, devops F1, perf F1, reviewer F4 — the CRITICAL, now resolved by decision 2]`.
- **Quarantine-child (2c) egress** `[devops F2, HIGH]`: the bwrap quarantine child's provider call routes through the gateway forward-proxy (provider-host-only allowlist); reconcile with #230 go-live + the 2b0 adversarial single-spawn-site allowlist.
- **Egress idempotency** `[security F4, comms F4, error F2, perf F4, test F1/F2]`: (i) egress-id must be a **deterministic** function of `(committed inbound-id, session, per-turn egress ordinal)` — a `uuid4` per call silently double-fires on a re-run; (ii) the dedup record must be **core-stamped + core-committed durable** (keep the gateway stateless; a gateway restart must not re-enable double-send); (iii) on a duplicate, **memoize + replay the stored response** flagged `deduplicated` — never silently drop (silent loss at a money boundary); (iv) **bound** the dedup set (TTL = the replay window); (v) downgrade the language from "exactly-once external call" to "**at-most-once at the gateway + egress-id forwarded as the remote idempotency key** where the API supports one" (true exactly-once needs the remote endpoint's cooperation); (vi) add a fake-external-world seam + a deterministic *egress* barrier (kill after the call commits, before the response is acked) for the release-blocking test.
- **Allowlisted-destination exfil** `[security F3]`: "DLP-in-core, allowlist-in-gateway" gates **destination**, not content over allowlisted channels — a compromised core can exfil in the body the gateway forwards verbatim. Honest scope in ADR-0034. The real second line is **mandatory canary-scan-on-egress, per-destination-independent** (the one content check the payload-blind gateway performs); resolve the perf tension (`[perf F3]`: canary-scan *is* reading the body) as a bounded/streamed envelope+canary scan, and correct decision-5's "does not re-inspect" wording.
- **Kernel-cap-vs-proxy regression** `[security NS-4, HIGH]`: today egress is capped by the sandbox **kernel** per-plugin (stronger than a userspace allowlist). ADR-0034 must (1) prove the network-namespace split is the enforcement-of-record (not the proxy allowlist), (2) decide whether the per-plugin kernel cap stays as defense-in-depth *behind* the proxy (removal = weakening a HARD default), (3) carry cred-concentration to the corpus before G7.
- **Network split topology** `[devops F4]`: two custom networks — `alfred_internal` (`internal: true`, core↔datastores↔gateway-socket) + `alfred_external` (gateway only). Core joins only internal. Handle the Postgres host-port-publish interaction; the gateway/forward-proxy must exist **before** the core loses its net (G3-before-G7 sequencing).
- **Egress-response quarantine path** `[comms F3, HIGH]`: an egress response (T3 tool result) **cannot** reuse `process_inbound_message` (no platform_user_id/persona/per-user rate). Define a separate quarantine-extract entry keyed on the turn's `canonical_user_id` + tool-call id.
- **Core fail-loud when the gateway is down** `[error F1/F3]`: connectivity-free core + gateway-unreachable = total external-I/O outage → a typed `IOPlaneUnavailableError` (loud, audited, distinct from `EgressDeniedError`), bounded timeout (no hang), `core_gateway_link_up` metric. Egress-deny → distinct `EgressDeniedError` surfaced to the agent + operator (not a generic tool failure), reason via `t()`.
- **HoL blocking** `[perf F2, HIGH]`: all external I/O on one event loop — every egress call an isolated task (TaskGroup) with a per-call timeout + per-destination/global concurrency cap; the egress path must not share a serial await chain with the comms relay; `gateway_egress_inflight` gauge + saturation alert.
- **Connectivity-free enforceable test** `[test F3, HIGH]`: assert `EgressClient` has no socket-opening capability + an integration test that a direct core→external connect **fails loud** in the split topology + an import-guard lint (no in-core http/socket client outside the wire client). A structural claim with no test rots on the first plugin that re-opens a socket.
- **PRD invariant timing** `[architect NS-6, reviewer F2]`: the "connectivity-free / all-I/O-via-gateway" PRD §5 invariant lands **only with Spec C / G7**, never earlier (it is false through A/B). Land the diagram earlier as "target architecture," promote to invariant at G7. Also fix the §5 diagram's stale Redis-streams comms-bus depiction (comms ingress is the notification model).
- **Citation fix** `[reviewer F5]`: the draft cited "PRD §49" — that's line 49 = **goal #8** (adversarial robustness); the egress policy is **§7.1 Egress allowlists** (line 445). Fix everywhere before ADRs lift it.

## ADRs (B/C)

- **ADR-0034** — Connectivity-free core / mandatory egress chokepoint: the network split (enforcement-of-record), the two egress modes, provider forward-proxy, egress idempotency contract, credential-concentration mitigation, kernel-cap disposition, honest exfil-scope. Human-gated.
- **PRD §5 rewrite** — gateway above the core as the I/O plane; the connectivity-free invariant (lands with C); centralize §7.1 egress enforcement at the gateway; fix the comms-bus drift. Human-gated.
