# G7-2.5 — Live `web.fetch` Re-home Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Re-home the live `web.fetch` tool egress so `dispatch_web_fetch` performs the fetch through the gateway `EgressRelay` (via the shipped C1/C2 relay machinery) as a fused fetch+extract unit returning a T2 `EgressExtractOutcome`, instead of spawning the `alfred_web_fetch` subprocess.

**Architecture:** The connectivity-free core hands a redacted GET `_RawToolRequest` to `EgressResponseExtractor` (C2) → `RelayEgressClient` (C1) → gateway `EgressRelay` over the framed JSON wire. The gateway fetches + returns raw T3 bytes; the core mints the `ContentHandle`, runs a web.fetch-specific pre-extract inspection (MIME/size/inbound-canary), extracts via the one gate-checked `quarantined_to_structured` seam, and records the post-extraction T2 in the §5 dedup ledger. `dispatch_web_fetch` keeps its core-side gatekeeping (URL/header DLP, 3-way path-prefix allowlist, rate-limit, success audit) and drops the machinery the gateway now owns (SSRF/redirect/TLS/size-origination + the subprocess).

**Tech Stack:** Python 3.12+, asyncio, Pydantic v2, SQLAlchemy 2.0 + Alembic, Postgres 16 (testcontainers), pytest + hypothesis + the adversarial harness, structlog, `t()` i18n (pybabel), Docker Compose.

## Task 1: Fold method+URL+schema identity into the dedup integrity hash (C6)

**Why:** for a GET, `_RawToolRequest.body=""`, so C1's `body_hash = compute_body_hash("")` is a constant → the `EgressIdIntegrityError` tripwire is blind to a divergent URL/schema replayed at the same `(ctx, call_index)`. Fold a canonical request descriptor into the hash so divergence fires loud.

**Files:**
- Modify: `src/alfred/egress/egress_id.py`
- Modify: `src/alfred/egress/relay_client.py:191-208` (the `scan_for_outbound` → `body_hash` block)
- Modify: `src/alfred/egress/egress_response_extract.py:140-168` (`handle` passes the descriptor)
- Test: `tests/unit/egress/test_egress_id.py`, `tests/unit/egress/test_relay_client.py`, `tests/unit/egress/test_egress_response_extract.py`

**Interfaces:**
- Produces: `compute_request_descriptor(*, method: str, url: str, schema_id: str) -> str` — sha256 hex over the length-prefixed `(method, url, schema_id)` (mirrors `compute_egress_id`'s encoding; injective field boundaries).
- Modifies: `RelayEgressClient.fire(self, *, raw_request, ctx, call_index, request_descriptor: str = "")`. `body_hash = compute_body_hash(request_descriptor + redacted_text)` (descriptor is fixed-width hex → no separator collision; default `""` preserves the synthetic-driver hash).
