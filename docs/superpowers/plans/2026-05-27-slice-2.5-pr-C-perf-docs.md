### Task 2 — The dispatch-overhead bench (delta-over-baseline)

**Owner:** `alfred-test-engineer`. TDD: write each bench, run it, read the reported p99, assert the budget. No `assert True` — the budget assertion is the test.

- [ ] **Step 2.1 — `tests/perf/__init__.py` + `tests/perf/conftest.py`.** Package marker + the shared harness:
  - `_run` — a module-level `asyncio.Runner` reused across all benches and the baseline (so the bench measures dispatch, not loop construction).
  - `baseline_delta` — a helper that, given a `pytest-benchmark` result for the measured callable and a separately-benchmarked no-op baseline, computes `p99(measured) - p99(baseline)` and returns the delta. Pure function over the two stats objects.
  - Registry fixtures reusing PR-A's `fresh_registry()` (import, don't re-implement). Each bench runs against a freshly-constructed registry so cross-bench subscriber leakage is impossible.
  - **Pinned bench knobs** applied per-bench via `benchmark.pedantic(..., rounds=…, iterations=…, warmup_rounds=…)`: fixed `rounds` and `iterations` (no auto-calibration — auto-calibration makes the p99 non-reproducible across CI hardware), `warmup_rounds >= 1`. The p99 is read off `benchmark.stats.stats` (pytest-benchmark exposes percentiles). Document the exact numbers chosen in the module header with the reasoning (high enough `iterations` that per-call noise averages out; high enough `rounds` that the p99 is stable).
- [ ] **Step 2.2 — The same-loop no-op baseline bench (`test_baseline_noop_delta_floor`).** Benchmarks `await _noop()` where `_noop` is a bare `async def _noop(): return None`, on the same `_run`. This is the floor every other bench subtracts. It is NOT itself budget-gated (it is hardware-absolute by design); it exists so the deltas are hardware-relative. Tripwire (concrete, not prose — TE-3): `import math; assert math.isfinite(p99) and p99 > 0.0` — a `0.0` from a mis-wired no-op or a `NaN` both fail loudly, keeping the bench off the "`assert True` placeholder" line.
- [ ] **Step 2.3 — Empty-hookpoint bench at a real publisher call site (`test_empty_hookpoint_dispatch_overhead`).** Construct an `EpisodicMemory` with `_persist` stubbed to a no-op coroutine and a **fresh empty registry** (zero subscribers). Benchmark the full `record(...)` body **minus the DB write** (the stub makes `_persist` free). Compute the delta over the Step 2.2 baseline. **Assert `p99_delta < 10e-6` seconds (10µs)** (spec §5). The failure message reports the measured delta and the budget so a regression is informative, not a bare `AssertionError`.
- [ ] **Step 2.4 — Informational HookContext+uuid4 micro-bench (`test_context_construction_micro_bench`).** Benchmarks `HookContext(action_id=…, hookpoint=…, input=…, correlation_id=str(uuid4()), kind="pre")` construction in isolation. **NOT gated** — it reports the intrinsic publisher cost (uuid4 + frozen-dataclass construction) separately from the dispatch engine, per spec §5 ("informational, not gated — it is intrinsic to the publisher, not the dispatch engine"). The test's only tripwire is concrete (TE-3): `import math; assert math.isfinite(p99) and p99 > 0.0` (a `0.0`/`NaN` from a mis-wired runner fails loudly); it prints the value via the benchmark report and carries a docstring stating it is informational.
- [ ] **Step 2.5 — 5-subscriber pre-chain bench (`test_five_subscriber_pre_chain_overhead`).** Register five `async` pass-through `pre` subscribers (each `return ctx`) on `memory.episodic.record.before_validate` against a `fresh_registry()` (operator/user-plugin tiers — `DevGate` grants those freely, so no `allow_system` needed; spec §6.2). Benchmark the same `record(...)`-minus-DB call. Compute the delta over baseline. **Assert `p99_delta < 100e-6` seconds (100µs)** (spec §5). Informative failure message.
- [ ] **Step 2.6 — Refusal short-circuit correctness assertion (`test_refusal_short_circuits_subscribers`).** NOT a bench — a paired correctness test (spec §5 pairs it with the bench). Register a `pre` chain on `before_validate` where subscriber #2 raises `HookRefusal` and subscribers #3–#5 each increment a shared counter. Invoke; assert (a) `HookRefusal` propagates, (b) the counter is `0` (subscribers after the refuser did not run), (c) `record`'s `_persist` was never called. This proves the short-circuit the perf budget assumes (an un-short-circuited chain would inflate the 100µs measurement and mask a correctness bug).
- [ ] **Step 2.7 — Run the perf suite locally and read the numbers.** Run **both** commands: (1) `uv run pytest tests/perf -v --benchmark-only` (the budget benches) and (2) `uv run pytest tests/perf -v -k refusal_short_circuits_subscribers` (the Step 2.6 correctness assertion). `--benchmark-only` deselects every test that does not use the `benchmark` fixture, so the refusal short-circuit check is **skipped** by command (1) and MUST be run separately — otherwise it never executes in the gate. Confirm all benches run, the two budget assertions pass on the implementer's hardware, the refusal-short-circuit correctness test passes, and the informational micro-bench + baseline produce sane numbers. If a budget fails on reasonable hardware, do NOT loosen the budget to make it pass (CLAUDE.md: do not weaken to pass) — investigate whether PR-A's dispatch path regressed and escalate.
- [ ] **Step 2.8 — Commit.** `test(perf): hook dispatch-overhead gate (empty <10µs, 5-chain <100µs, delta-over-baseline) (#104)`.
