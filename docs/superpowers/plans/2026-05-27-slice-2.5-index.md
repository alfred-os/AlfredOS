# Slice 2.5 — Implementation Plan Index

> **Slice 2.5 = pluggable hooks for every action.**
> Spec: [`docs/superpowers/specs/2026-05-27-slice-2.5-hooks-design.md`](../specs/2026-05-27-slice-2.5-hooks-design.md) (PR #112 — through `/review-pr` + `/review-plan` + CR-approved).
> ADR: [`docs/adr/0014-pluggable-hooks-for-every-action.md`](../../adr/0014-pluggable-hooks-for-every-action.md) (PR-C flips it Proposed → Accepted).
> Plans below are sequenced; each PR's plan lists what the next may assume.

## What ships

When all 3 PRs below have merged:

- A first-class **hookpoint** primitive in `src/alfred/hooks/` — named, string-keyed extension points any code (core or, from Slice 3, plugin) can both **publish** (`invoke()` / the `invoking()` helper) and **subscribe to** (`@hook`).
- Four hook kinds (`pre` / `post` / `error` / `cancel`) with deterministic tier-ordered dispatch (`system` → `operator` → `user-plugin`).
- A `CapabilityGate` Protocol + **`DevGate`** (refuses `system` tier unless constructed with `allow_system=True` — no ambient env flag).
- A fail-loud fault model: `HookRefusal` (pre short-circuit), `HookSubscriberError` (unexpected subscriber raise), per-chain `asyncio.timeout()` deadline with per-hookpoint `fail_closed`, re-entry guard, all audit-correlated via an `AuditSink`.
- One instrumented proof-of-concept action — **`memory.episodic.record`** with five hookpoints across all four kinds, including a system-tier `before_db_write` redaction demo.
- A release-blocking `pytest-benchmark` dispatch-overhead gate (<10µs empty / <100µs 5-hook, delta-over-baseline) + 100% line+branch coverage on the subsystem.
- `docs/subsystems/hooks.md` (hello-world-first), glossary entries, ADR-0014 accepted + amended, and a **separate human-gated proposed PRD §5.1 diff** (PRD edits are not made in a code commit).

**No behaviour change** to existing actions: `memory.episodic.record` with zero subscribers persists a byte-identical row (golden-row regression test).

### Sequencing

Strictly sequential: `A → B → C`. PR-A ships the full contract (registry, dispatch, gate, fault model, `AuditSink` Protocol, `invoking` helper) and is **not** blocked on PR-B (its coverage runs against the default structlog-only sink). PR-B wires the real `audit.append` sink + the PoC. PR-C benches the dispatch overhead (needs PR-A) at a real publisher call site (needs PR-B's `record()` body) and writes the docs.

```
A ──▶ B ──▶ C
```
