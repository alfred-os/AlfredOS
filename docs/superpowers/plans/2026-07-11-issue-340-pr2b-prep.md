## Scope note — P1c (cost channel) is deferred to PR2b-golive

The ratified spec §4 listed **P1c (cost sums across attempts + rides refusals + turn-level aggregation)** as a prep item. **Plan-time decision: move P1c to golive.** Cost is identically `None`/`0` until the real LLM runs, and surfacing it requires a **live wire-format change** (a new field on the `Extracted`/`TypedRefusal` result the child emits over fd-1 + the host model that parses it) plus a real turn-aggregation consumer — none of which can be behavior-neutral or has a prep-time purpose. Doing it here would be plumbing carrying nulls + a wire change with no consumer (premature; not behavior-neutral). Golive delivers cost where the value is real and the aggregation seam has a concrete owner (§19-D2). **This is the one deviation from the ratified §4 prep list — flagged for the requester; the other four items (P1a, P1b, P1d, P1e) are in this plan.**

```python
    def _lift_above_targets(fd: int) -> tuple[int, bool]:
        """Return ``(usable_fd, moved)``. Dup ``fd`` above the target range if it collides.

        Looping (not a single dup) covers the pathological case where the FIRST dup also
        lands on the (other) target. Each intermediate dup THIS function created is closed
        before the next iteration (P1d, #340): only the caller's original survives to the
        cleanup loop (closed there under ``moved``). Without this close a ≥2-iteration lift
        orphans the first intermediate fd.
        """
        moved = False
        while fd in literal_targets:
            new_fd = os.dup(fd)
            if moved:
                # ``fd`` is an intermediate WE created last iteration (not the caller's
                # original) — close it so it isn't orphaned. The original is closed by the
                # cleanup loop under ``moved=True``; closing it here would double-close.
                os.close(fd)
            fd = new_fd
            moved = True
        return fd, moved
```
