## Scope note — P1c (cost channel) is deferred to PR2b-golive

The ratified spec §4 listed **P1c (cost sums across attempts + rides refusals + turn-level aggregation)** as a prep item. **Plan-time decision: move P1c to golive.** Cost is identically `None`/`0` until the real LLM runs, and surfacing it requires a **live wire-format change** (a new field on the `Extracted`/`TypedRefusal` result the child emits over fd-1 + the host model that parses it) plus a real turn-aggregation consumer — none of which can be behavior-neutral or has a prep-time purpose. Doing it here would be plumbing carrying nulls + a wire change with no consumer (premature; not behavior-neutral). Golive delivers cost where the value is real and the aggregation seam has a concrete owner (§19-D2). **This is the one deviation from the ratified §4 prep list — flagged for the requester; the other four items (P1a, P1b, P1d, P1e) are in this plan.**

### Task 3: fix the `_lift_above_targets` ≥2-iteration intermediate-fd orphan — P1d

`_lift_above_targets` dups above the target range in a loop but never closes an intermediate dup when the loop iterates ≥2× (a source landing on the *other* target). Triple-unreachable today, but it runs on **every live spawn** (the `control_fd=False` single-target path). **Keep** the cleanup loop's `os.close(original)` under `moved=True` (`quarantine_child_io.py:759`) — removing it introduces a leak (the PR2a "extra live ref" concern was a proven misdiagnosis).

This restructures LIVE clobber-window code — copy the logic faithfully; the existing spawn suite is the byte-identity guard for the live `control_fd=False` path.

```python
def _lift_above_targets(
    fd: int,
    literal_targets: tuple[int, ...],
    *,
    dup: Callable[[int], int] = os.dup,
    close: Callable[[int], None] = os.close,
) -> tuple[int, bool]:
    """Return ``(usable_fd, moved)``. Dup ``fd`` above the target range if it collides.

    Looping (not a single dup) keeps the invariant "the returned fd is never one of
    ``literal_targets``" true unconditionally. Each intermediate dup THIS function created
    is closed before the next iteration (P1d, #340): only the caller's ORIGINAL survives to
    the spawn cleanup loop (closed there under ``moved``). Without this in-loop close a
    ≥2-iteration lift (a source landing on the OTHER target) orphans the first intermediate.

    ``dup``/``close`` are injectable so the ≥2-iteration branch is unit-testable without a
    real ``dup2``-onto-3/4 (which would clobber the pytest runner).
    """
    moved = False
    while fd in literal_targets:
        new_fd = dup(fd)
        if moved:
            # ``fd`` is an intermediate WE created last iteration (not the caller's
            # original) — close it so it isn't orphaned. The spawn cleanup loop closes the
            # original under ``moved=True``; closing it here would double-close.
            close(fd)
        fd = new_fd
        moved = True
    return fd, moved
```

### Task 4: reconcile the timeout hierarchy + add the ordering-invariant guard — P1e

As shipped the constants are inverted (host read-frame 15 < child budget 30 < SDK read 60), which would tear a real extraction host-side. Fix the inversion and set monotone values: **`action_deadline (30) ≥ host read-frame (25) ≥ child wall-clock budget (20)`**.
