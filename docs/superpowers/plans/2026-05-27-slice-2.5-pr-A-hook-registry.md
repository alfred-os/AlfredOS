# Slice 2.5 — PR A: Hook registry + contract — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development`. Steps use checkbox (`- [ ]`) syntax — tick them as you go so a follow-up session can resume without re-deriving state.

**Goal:** Land the `src/alfred/hooks/` subsystem — the named-hookpoint contract and dispatch engine — with zero actions wired. After this PR merges, any in-tree module can `@hook(...)`-register an async subscriber against a `DevGate`-enforced tier, and any publisher can `await invoke(name, ctx, kind=...)` or `async with invoking(action_id, input) as flow` to dispatch a deterministic, capability-gated, per-chain-deadlined chain across all four kinds (`pre`/`post`/`error`/`cancel`). The no-subscriber path is a single dict lookup returning the `_EMPTY` tuple singleton with an early return before the re-entry ContextVar is read; a slow or buggy subscriber is cancelled-to-completion and the hookpoint's `fail_closed` policy decides refuse-vs-pass-through; a sync function passed to `@hook` is rejected at decoration time; and the whole subsystem ships under a 100% line+branch coverage gate because it is a trust boundary. The PoC action (PR-B) and the perf-bench suite + subsystem docs (PR-C) are out of scope here.

**Spec:** [`docs/superpowers/specs/2026-05-27-slice-2.5-hooks-design.md`](../specs/2026-05-27-slice-2.5-hooks-design.md) — this PR implements §3 (architecture, all of `src/alfred/hooks/`), §4 (the four kinds + dispatch sequence), §5 (performance — the registry/dispatch side: `_EMPTY` no-alloc path, the per-chain `asyncio.timeout()`, no per-subscriber timer), §6 (the full security contract — §6.2 `DevGate`, §6.3 two-gate model, §6.4 chain timeout + fail-closed, §6.5 refusal authorization, §6.6 unexpected subscriber exceptions, §6.8 audit correlation, §6.9 re-entry detection), and §8 (the testing strategy + 100% coverage gate). It does **not** implement §7 (the `memory.episodic.record` PoC — that is PR-B), the perf-bench suite / `pytest-benchmark` gate / §10 docs / §9 ADR+PRD edits (those are PR-C). The locked v4 type contract this plan reproduces verbatim is §3.3 (`HookContext`), §3.4 (`invoke`/`invoking`), §3.5 (`@hook`), and the §12 PR-A scope line.

**Depends on:** `main` as of Slice 2 (Slice 2 graduated; PR #108 is the last commit). Concretely: `src/alfred/errors.py` `AlfredError` (Slice 1), `src/alfred/i18n/translator.py` `t()` + per-coroutine `_active_lang` ContextVar (Slice 2 PR A), and the `locale/en/LC_MESSAGES/alfred.po` catalog + `pybabel` pre-commit/CI gates (Slice 1/2). No dependency on any Slice 2.5 PR — PR-A is the contract everyone else codes against.

**Architecture:** A new top-level subpackage `src/alfred/hooks/` with seven source modules behind one `__init__.py` public surface (§3 of the spec). The split is deliberate so each file's branches are independently coverable under the 100% gate:

- **`context.py`** — `HookContext[T]` frozen-slots generic dataclass (PEP 695 generic, Python 3.14) with `with_input` / `with_metadata` / `for_stage` copy helpers (frozen → every helper returns a NEW instance via `dataclasses.replace`). `type HookKind = Literal["pre","post","error","cancel"]` lives here so both `invoke` and `decorators` import it from one place.
- **`errors.py`** — `HookError(AlfredError)` → `{HookRefusal, HookSubscriberError}`, each carrying its operator-facing message through `t()`. `HookRefusal` carries `hook_id`/`action_id`/`reason`/`correlation_id` keyword-only; `HookSubscriberError` wraps an unexpected subscriber exception and chains it via `raise ... from`.
- **`capability.py`** — `CapabilityGate` Protocol (`check(*, plugin_id, hookpoint, requested_tier) -> bool`) + `DevGate` (Slice-2.5 default). `DevGate.allow_system` is **constructor-only** (`DevGate(allow_system=True)`); there is **no env flag** (sec-007). `DevGate` grants `operator`/`user-plugin` freely and refuses `system` unless opted in.
- **`audit_sink.py`** — `AuditSink` Protocol (`async emit(*, event, correlation_id, fields)`) + a default **structlog-only** sink (no DB; PR-B wires the real `audit.append`-backed sink). PR-A's coverage runs against this structlog sink (arch-001).
- **`registry.py`** — `HookRegistry` instance + `get_registry()`/`set_registry()`. Keyed by `(hookpoint, kind)`, ordered by `(_TIER_RANK[tier], registration_seq)`, returning a frozen tuple; the empty case returns the module-level `_EMPTY: tuple[Subscriber, ...] = ()` singleton (object-identity asserted). Records each subscriber's `origin_module`. Owns `reset()` (test isolation) and the per-task re-entry ContextVar stack. `HOOK_CHAIN_DEADLINE_SECONDS = 0.25` and `_TIER_RANK = {"system":0,"operator":1,"user-plugin":2}` live here.
- **`decorators.py`** — `@hook(name, *, kind, tier)`. A sync function is rejected **at decoration time** with `HookError(t("hooks.subscriber_must_be_async", name=fn.__qualname__))`. The decorator registers against `get_registry()` (the import-time fallback) so module-level `@hook` resolves to *some* registry.
- **`invoke.py`** — the `invoke[T]` primitive (authoritative for `kind`/`hookpoint`: it applies `ctx.for_stage(...)` before dispatch), four private handlers (`_run_pre`/`_run_post`/`_run_error`/`_run_cancel`) over a shared `_run_chain` core, the per-chain `asyncio.timeout(deadline)` (deadline **injectable** for tests — test-101), the cancelled-subscriber-awaited-to-completion logic placed in the `except TimeoutError` handler (core-006), the re-entry guard via the registry ContextVar with a core-private `_invoke_internal` bypass unreachable from subscribers (sec-008), and the `invoking()` async-context-manager helper that mints the `correlation_id`, threads the frozen `ctx`, and fires the `cancel` chain **before** the `error` chain on `CancelledError` (test-102).

The registry holds **no per-invocation state**; invocation state (the correlation-ID re-entry stack) lives in a `ContextVar` so it is per-task, mirroring `src/alfred/i18n/translator.py`'s `_active_lang` pattern. No global mutable state: `get_registry()` lazily constructs a process-default instance and `set_registry()` swaps it (a bootstrap-and-test seam this slice — **not** hot-reload; arch-002).

**Tech Stack:** Python 3.14 (repo `requires-python = ">=3.14"`; PEP 695 generics `class HookContext[T]` / `async def invoke[T]` / `type HookKind = ...` are first-class) • `dataclasses` (frozen + slots) • `asyncio` (`asyncio.timeout`, `contextvars.ContextVar`, structured concurrency where a chain spawns nothing) • `typing.Protocol` for `CapabilityGate`/`AuditSink` • `structlog` (slice-1 dep) for the default `AuditSink` • `alfred.errors.AlfredError` root • `alfred.i18n.t()` for every operator-facing message • pytest + `hypothesis` (where a one-sentence property holds — e.g. tier-then-seq ordering, idempotence of `for_stage`) • coverage with per-file `--cov-branch --cov-fail-under=100` on `src/alfred/hooks/{registry,invoke,capability,decorators,context,errors,audit_sink}.py`. **No `pytest-benchmark`** in this PR (PR-C dep).

**Subagent owner:** `alfred-core-engineer` — owns the OODA/orchestrator-adjacent dispatch spine, the plugin contract, async-first discipline, and no-global-state. **`alfred-security-engineer` co-reviews the gate + fault tasks** (Tasks 3, 4, 9, 10, 11, 12, 13 — `DevGate` refusal + the sec-007 no-env-read guard, two-gate model, chain-timeout fail-closed, unexpected-subscriber-exception fault policy, unauthorized-refusal handling, re-entry bypass) because each sits on the in-band capability boundary; co-review is the rule-#4 / rule-#7 / rule-#13 gate (no gate stubbed to always-allow; no silent failures in security paths). **`src/alfred/hooks/__init__.py` (new).** The public surface — the contract everyone codes against (§3.1). Re-exports exactly: `hook`, `invoke`, `invoking`, `HookContext`, `HookKind`, `HookError`, `HookRefusal`, `HookSubscriberError`, `CapabilityGate`, `DevGate`, `AuditSink`, `HookRegistry`, `get_registry`, `set_registry`. **`src/alfred/hooks/context.py` (new).** `type HookKind = Literal["pre","post","error","cancel"]`. `HookContext[T]` reproduced **verbatim** from §3.3 (the locked v4 contract):

```python
@dataclass(frozen=True, slots=True)
class HookContext[T]:
    action_id: str
    hookpoint: str
    input: T
    correlation_id: str
    kind: Literal["pre", "post", "error", "cancel"]
    metadata: Mapping[str, object] = field(default_factory=dict)

    def with_input(self, new_input: T) -> Self:
        return replace(self, input=new_input)

    def with_metadata(self, **kv: object) -> Self:
        return replace(self, metadata={**self.metadata, **kv})

    def for_stage(self, *, hookpoint: str, kind: HookKind) -> Self:
        """Return a NEW context retargeted to a stage. Frozen — never mutates."""
        return replace(self, hookpoint=hookpoint, kind=kind)
```

`metadata` default is `field(default_factory=dict)` — a per-instance empty dict, never a shared mutable default. `Self` (PEP 673) is the return type so subclass-friendliness is free; `Mapping[str, object]` (not `Any`, not a typed-per-hookpoint param — YAGNI per spec §13). **Forward-compat reservation comment** (spec §13): a one-line docstring note that a future value-returning-action carrier adds `output: TOutput | None` here (or `invoke` gains `result=` mirroring `exc=`) — no field this slice, but the name is reserved so the carrier lands without churning `invoke`'s signature.

**`src/alfred/hooks/errors.py` (new).** The hook exception taxonomy rooted at `AlfredError` (via `alfred.errors`):

```python
class HookError(AlfredError): ...

class HookRefusal(HookError):
    def __init__(self, *, hook_id: str, action_id: str, reason: str, correlation_id: str) -> None: ...

class HookSubscriberError(HookError): ...   # wraps an unexpected subscriber exception, chains via `from`
```

`HookRefusal.__init__` is keyword-only, stores all four fields as attributes (for the audit row + caller introspection), and builds its message via `t("hooks.refusal", hook_id=..., action_id=..., reason=...)`. `HookSubscriberError` is raised as `raise HookSubscriberError(t("hooks.subscriber_error", name=..., correlation_id=...)) from exc` so the original traceback chains (rule #13 — never silently swallowed). The sync-rejection message `t("hooks.subscriber_must_be_async", name=...)` is raised as a plain `HookError` from `decorators.py` (Task 7). All three messages are operator/developer-facing → `t()` (i18n rule #1).

**`src/alfred/hooks/capability.py` (new).** The two-gate security primitive (§6.2, §6.3):

```python
class CapabilityGate(Protocol):
    def check(self, *, plugin_id: str, hookpoint: str, requested_tier: str) -> bool: ...

class DevGate:
    def __init__(self, *, allow_system: bool = False) -> None: ...
    def check(self, *, plugin_id: str, hookpoint: str, requested_tier: str) -> bool: ...
```

`DevGate.check` returns `True` for `operator`/`user-plugin` always; for `system` it returns `self._allow_system`; for any unknown tier string it returns `False` (fail-closed on a typo'd tier). `allow_system` is set **only** through the constructor — there is no `os.environ` read anywhere in this module (sec-007; an AST-scan test in Task 4 proves the module reads no env). `CapabilityGate` is a `@runtime_checkable` Protocol so `isinstance(DevGate(), CapabilityGate)` holds in a test.

**`src/alfred/hooks/audit_sink.py` (new).** The dispatch-engine's audit write seam (§6.8, §3.1):

```python
class AuditSink(Protocol):
    async def emit(self, *, event: str, correlation_id: str, fields: Mapping[str, object]) -> None: ...
```

**Canonical `emit` call form (keyword-only — every call site in this PR and PR-B uses exactly this shape).** `emit` is fully keyword-only (the `*` in the Protocol); `event` is **not** positional. Every dispatch/registry call site reads:

```python
await sink.emit(
    event="hooks.subscriber_error",
    correlation_id=ctx.correlation_id,
    fields={...},
)
```

A positional `event` (`emit("hooks.refusal", ...)`) is a `TypeError` under the `*` signature and fails mypy/pyright strict (alfred-reviewer-1). Tasks 6/10/11/12 below — and PR-B's `EpisodicAuditSink` calls — all match this canonical form verbatim.

Plus `StructlogAuditSink` — the Slice-2.5 default — whose `emit` binds `event`/`correlation_id`/`fields` onto a module-level structlog logger and `.info(...)`s a hook-trace row. **No DB write** (PR-B wires the real `audit.append`-backed sink; arch-001 — PR-A coverage runs entirely against this structlog sink and is NOT blocked on PR-B). The events this sink must accept (string constants defined here so dispatch and tests share one source): `hooks.refusal`, `hooks.chain_timeout`, `hooks.subscriber_error`, `hooks.error_suppressed`, `hooks.unauthorized_refusal`, `hooks.reentry_bypass`. The fields the sink redacts: it routes any `fields` value through the structlog redactor chain already configured in `cli/main.py` (no raw secret in a hook-trace row — rule #1/#2). The default sink is a frozen-config dataclass holding the logger so it is injectable (no module global).

**`src/alfred/hooks/registry.py` (new).** The registry + its module-level constants:

```python
HOOK_CHAIN_DEADLINE_SECONDS = 0.25          # per-CHAIN deadline (one asyncio.timeout per chain)
_TIER_RANK = {"system": 0, "operator": 1, "user-plugin": 2}
_EMPTY: tuple[Subscriber, ...] = ()         # module singleton; identity-asserted in a test
```

`Subscriber` is a frozen dataclass: `(hook_fn, hookpoint, kind, tier, origin_module, registration_seq)`. `HookRegistry`:
- `__init__(self, *, gate: CapabilityGate, sink: AuditSink = StructlogAuditSink())` — the gate **and the audit sink** are injected (no global; production threads the real gate + sink at bootstrap, tests pass `DevGate(...)` + a spy sink). The `sink` is the **AuditSink injection seam** the whole dispatch engine writes hook-trace rows through (resolves alfred-core-engineer-1): `_run_chain` reaches it via `get_registry().sink` (or the injected registry's `.sink`), so the locked `invoke()`/`invoking()` signatures gain **no** `sink=` parameter — they read the sink off the registry that owns them. The `spy_sink` fixture injects by constructing a registry with `sink=spy`. `sink` is a public attribute (`self.sink`) so dispatch and tests read it uniformly. Holds `_subscribers: dict[tuple[str, HookKind], list[Subscriber]]`, a monotonic `_seq` counter, and the per-task `_reentry: ContextVar[tuple[str, ...]]` re-entry stack (default `()`).
- `register(self, *, hook_fn, hookpoint, kind, tier) -> None` — validates async-only (raise `HookError(t("hooks.subscriber_must_be_async", ...))` if not a coroutine function — shared with the decorator), consults `self._gate.check(...)` (refused → `HookError` + `await self.sink.emit(event="hooks.refusal", correlation_id=..., fields={...})` — fail-loud, not silent, canonical keyword-only emit form), assigns `registration_seq`, records `origin_module = hook_fn.__module__`, inserts keyed by `(hookpoint, kind)`, and keeps the list sorted by `(_TIER_RANK[tier], registration_seq)`.
- `subscribers_for(self, hookpoint, kind) -> tuple[Subscriber, ...]` — O(1) dict lookup. Returns the frozen tuple; the miss returns `_EMPTY` (same object every call — identity-tested for the no-alloc proof).
- `reset(self) -> None` — clears `_subscribers` + `_seq` for test isolation.
- `get_registry() -> HookRegistry` / `set_registry(reg: HookRegistry) -> None` — module-level accessor/swap. `get_registry()` lazily constructs a default instance with a default `DevGate()` (system-refused) on first call. **arch-002 forward-compat note** (in the module docstring): which registry snapshot an in-flight chain resolves against during a `set_registry()` swap is a Slice-3 concern — there is no live reload this slice; today's invariant is only that dropping a registry instance drops its subscribers.

**`src/alfred/hooks/decorators.py` (new).** `@hook(name, *, kind, tier)` reproduced from §3.5:

```python
def hook(name: str, *, kind: HookKind, tier: Literal["system", "operator", "user-plugin"]): ...
```

The decorator inspects the decorated function with `inspect.iscoroutinefunction`; a sync function raises `HookError(t("hooks.subscriber_must_be_async", name=fn.__qualname__))` **at decoration time** (before any registry mutation). On success it calls `get_registry().register(hook_fn=fn, hookpoint=name, kind=kind, tier=tier)` and returns `fn` unchanged (so the decorated function is still directly callable in tests). The subscriber contract documented in the decorator docstring: a subscriber returns `HookContext[T] | None` — `None` means "no change / proceed with default disposition" (for `error` kind, `None` = re-raise).

**`src/alfred/hooks/invoke.py` (new).** The dispatch primitive + helper, reproduced from §3.4:

```python
async def invoke[T](
    name: str,
    ctx: HookContext[T],
    *,
    kind: HookKind,
    subscribable_tiers: frozenset[str] = frozenset({"system", "operator", "user-plugin"}),
    refusable_tiers: frozenset[str] = frozenset({"system", "operator", "user-plugin"}),
    fail_closed: bool = False,
    exc: BaseException | None = None,
) -> HookContext[T]: ...
```

`invoke` is **authoritative for kind/hookpoint**: it first applies `ctx = ctx.for_stage(hookpoint=name, kind=kind)` so subscribers always see the correct stage even if the caller passed a stale `ctx`. It then routes to one of four private handlers (`_run_pre`/`_run_post`/`_run_error`/`_run_cancel`) sharing a `_run_chain` core — each handler is a thin branch so the 100% gate can assert each kind's path independently. The deadline is **injectable** (`invoke` reads `HOOK_CHAIN_DEADLINE_SECONDS` by default but `_run_chain` takes a `deadline_seconds: float` parameter that tests drive to ~0; test-101) and the chain is wrapped in **one** `asyncio.timeout(deadline_seconds)` (no per-subscriber timer — §5/§6.4). `_run_chain` writes every fault row through the **registry-owned sink** — `get_registry().sink` (or the sink of the registry that owns the in-flight chain) — using the canonical keyword-only `await sink.emit(event=..., correlation_id=..., fields={...})` form; the `invoke()`/`invoking()` signatures take no `sink=` parameter (alfred-core-engineer-1). On `TimeoutError`, the in-flight subscriber's coroutine is **awaited to completion in the `except TimeoutError` handler** (core-006: awaiting inside the still-active `asyncio.timeout()` block re-cancels) so it cannot leak a half-open cursor/lock, then the `fail_closed` policy decides refuse-vs-pass-through (§6.4). An unexpected subscriber exception (neither `HookRefusal` nor timeout) is wrapped as `HookSubscriberError`, audited (`event="hooks.subscriber_error"`), and the `fail_closed` policy applied (§6.6). An unauthorized `HookRefusal` (tier ∉ `refusable_tiers`) is audited (`event="hooks.unauthorized_refusal"`), **ignored, and no exception propagates** (§6.5). The re-entry guard reads the registry's `_reentry` ContextVar; a re-entrant call to the same `(hookpoint)` within the same correlation chain dispatches via a **core-private `_invoke_internal`** path that skips the chain (all tiers, system included — T0-only invariant) and emits `hooks.reentry_bypass` (§6.9). `_invoke_internal` is module-private and never exported (sec-008). The `invoking(action_id, input, *, gate=None)` async context manager mints `correlation_id = str(uuid4())`, builds the initial frozen `HookContext`, exposes `flow.pre(stage, **invoke_kwargs)` / `flow.input` / `flow.body()`, fires the `cancel` chain **before** the `error` chain on `CancelledError` (test-102) and the `post` chain on success, and always re-raises `CancelledError`. **`flow.pre()` return contract (pinned — resolves alfred-core-engineer-1 / arch-001).** `await flow.pre(stage, **invoke_kwargs)` runs the `pre` chain for `stage` and returns the **same flow object** after threading the new frozen `HookContext` (the chain output) into the flow's internal `_ctx` holder — so `flow.input` immediately reflects the latest stage's chain output (e.g. a `before_db_write` subscriber's `with_input` mutation). The `HookContext` stays frozen (every chain step is a new `replace()`d instance); the **flow wrapper is the mutable holder** of the current ctx ref. Callers may therefore either `await flow.pre(...)` then read `flow.input`, or `flow = await flow.pre(...)` (same object) — both observe the threaded result. Task 13 adds a test asserting `flow.input` reflects a `pre`-subscriber's `with_input` mutation after `await flow.pre(...)`.

**`tests/unit/hooks/test_dispatch.py` (new).** The four-kind dispatch core (§4): a `pre` subscriber's `with_input` mutation flows to the action (the returned `ctx.input` reflects it); a `pre` mutation by an earlier subscriber is **discarded when a later subscriber refuses** (the action never sees the mutated input — asserted by capturing what `invoke` returns vs. raises); `post` mutation flows to downstream chain observers; `error` swallow-and-substitute (subscriber returns a context → `invoke` returns it, no re-raise) **and** re-raise (subscriber returns `None` → original `exc` re-raised, chained); `cancel` cleanup-only for both a normal-returning and a raising cancel subscriber (neither can suppress — the `CancelledError` always propagates); each of `_run_pre`/`_run_post`/`_run_error`/`_run_cancel` is exercised so its branch is covered; `invoke` applies `for_stage` (a caller passing a stale `hookpoint`/`kind` still gets the right stage in the subscriber — asserted by reading `ctx.hookpoint`/`ctx.kind` inside a spy subscriber).

## 4. What PR-B and PR-C may assume (the full public surface)

PR-A ships the entire `src/alfred/hooks` contract; PR-B (the `memory.episodic.record` PoC) and PR-C (perf bench + docs + ADR/PRD edits) code against it without re-deriving it. The load-bearing surface PR-A pins:

- **`src/alfred/hooks` public exports (`__all__`, §3.1).** `hook`, `invoke`, `invoking`, `HookContext`, `HookKind`, `HookError`, `HookRefusal`, `HookSubscriberError`, `CapabilityGate`, `DevGate`, `AuditSink`, `HookRegistry`, `get_registry`, `set_registry`.
- **`invoke[T](name, ctx, *, kind, subscribable_tiers=…, refusable_tiers=…, fail_closed=False, exc=None) -> HookContext[T]`** — authoritative for `kind`/`hookpoint` (applies `for_stage`). The locked v4 signature; will not change in PR-B/PR-C.
- **`invoking(action_id, input, *, gate=None)`** async context manager — `flow.pre(stage, **invoke_kwargs)`, `flow.input`, `flow.body()`. **`await flow.pre(stage, **kw)` returns the same flow object** and threads the `pre` chain's output frozen `HookContext` into the flow's internal `_ctx` holder, so `flow.input` reflects the latest stage's chain output (the ctx stays frozen; the flow wrapper is the mutable holder). PR-B's `record()` rewrites against exactly this surface — `await flow.pre(...)` then read `flow.input` — with **no "adapt to whatever shipped" hedge** (the contract is pinned here, Task 13). PR-A pins the `flow.pre`/`flow.body()` shape per spec §13's "PR-A pins the exact signatures" note.
- **`@hook(name, *, kind, tier)`** — subscriber decorator; subscriber returns `HookContext[T] | None`; sync function rejected at decoration time.
- **`HookContext[T]`** frozen-slots dataclass with `with_input`/`with_metadata`/`for_stage`; the `output` carrier is **reserved, not implemented** (§13) — PR-B's PoC returns `None` and needs no carrier.
- **`DevGate(*, allow_system=False)`** — PR-B's integration test registers the system-tier redactor against `DevGate(allow_system=True)` constructed explicitly (the granted-system path).
- **`AuditSink` Protocol + `StructlogAuditSink`** — PR-B replaces/extends the default sink with one backed by the real `audit.append` (the PoC is `memory.episodic.record`, not `audit.append`, so there is no recursion — §6.8). PR-A's coverage does NOT depend on PR-B's sink existing (arch-001).
- **`HOOK_CHAIN_DEADLINE_SECONDS = 0.25`** default + the injectable `deadline_seconds` seam — PR-C's perf-bench suite measures dispatch-overhead delta-over-baseline against the empty-chain `_EMPTY` path and the 5-subscriber `pre` chain.
- **The six audit event-name constants** (`hooks.refusal`, `hooks.chain_timeout`, `hooks.subscriber_error`, `hooks.error_suppressed`, `hooks.unauthorized_refusal`, `hooks.reentry_bypass`) — PR-B's adversarial `hooks` corpus category asserts the `tier_escalation` payload trips `hooks.refusal`/`hooks.unauthorized_refusal`.
